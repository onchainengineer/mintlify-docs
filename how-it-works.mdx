# How It Works

## Overview

wirtual.dev transforms physical AI development by providing instant simulation environments through a powerful yet simple architecture.

## Core Architecture

### 1. Control Layer
- **Environment Orchestrator**
 - Manages lifecycle of simulation environments
 - Handles template deployment
 - Controls resource allocation
 - Monitors system health

- **Template Engine**
 - Manages versioned templates
 - Handles environment configurations
 - Processes custom templates
 - Validates configurations

- **Resource Scheduler**  
 - Allocates compute resources
 - Manages GPU distribution
 - Handles scaling operations
 - Optimizes resource usage

### 2. Compute Layer
- **Environment Runner**
 - Executes simulation environments
 - Manages container lifecycle
 - Handles state persistence
 - Ensures isolation

- **GPU Manager**
 - Allocates GPU resources
 - Optimizes utilization
 - Manages sharing policies
 - Handles failover

- **Storage Controller**
 - Manages persistent volumes 
 - Handles data replication
 - Controls snapshot lifecycle
 - Optimizes I/O performance

### 3. Access Layer  
- **Authentication Service**
 - Handles user authentication
 - Manages access tokens
 - Controls session lifecycle
 - Integrates with SSO

- **Authorization Manager**
 - Enforces access policies
 - Manages role assignments
 - Controls resource limits
 - Validates permissions

- **Audit Logger**
 - Records system events
 - Tracks user actions
 - Monitors resource usage
 - Generates compliance reports
