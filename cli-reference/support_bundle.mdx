# support bundle

Generate a support bundle to troubleshoot issues connecting to a workspace.

## Usage

```console
wirtual support bundle [flags] <workspace> [<agent>]
```

## Description

```console
This command generates a file containing detailed troubleshooting information about the wirtual deployment and workspace connections. You must specify a single workspace (and optionally an agent name).
```

## Options

### -y, --yes

|      |                   |
| ---- | ----------------- |
| Type | <code>bool</code> |

Bypass prompts.

### -O, --output-file

|             |                                                |
| ----------- | ---------------------------------------------- |
| Type        | <code>string</code>                            |
| Environment | <code>$WIRTUAL_SUPPORT_BUNDLE_OUTPUT_FILE</code> |

File path for writing the generated support bundle. Defaults to wirtual-support-$(date +%s).zip.

### --url-override

|             |                                                 |
| ----------- | ----------------------------------------------- |
| Type        | <code>string</code>                             |
| Environment | <code>$WIRTUAL_SUPPORT_BUNDLE_URL_OVERRIDE</code> |

Override the URL to your wirtual deployment. This may be useful, for example, if you need to troubleshoot a specific wirtual replica.
